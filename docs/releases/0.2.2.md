# ProbRAW 0.2.2

Release focused on performance, interface fluidity and reproducibility.

## Main changes

- `batch-develop` uses real multiprocessing per process when there are several
  workers.
- New demosaic opt-in numerical cache (`use_cache: true`) to reuse
  the linear scene when only later settings change.
- New golden tests for canonical hashes and controlled regeneration script.
- New reproducible RAW pipeline and GUI interaction benchmarks.
- Final refresh of preview after dragging sliders or tone curve is queued
  in the background for large images when there is no ICC preview active.
- `write_tiff16` reduces NumPy temporals during conversion to TIFF16.
- Automatic worker heuristics reserve more memory per process
  (`2800 MiB`) for high resolution RAWs.

## Reference local results

Benchmark with RAW Nikon D850 8288x5520 51.5 MiB on Windows 11, Python 3.12.4
and 32 logical threads:

- demosaic `dcb`: ~5.36 s,
- demosaic `amaze`: ~5.57 s,
- `dcb` demo cache hit: ~0.16 s,
- `develop` complete with cache hit, final TIFF + audit: ~1.59 s,
- 4 `dcb` jobs with 4 workers: ~5.97 s vs. ~21.31 s serial.

GUI Benchmark with Full RAW 5520x8288:

- brightness slider: p95 event UI ~0.053 ms, p95 event loop ~16.72 ms,
- tone curve: p95 event UI ~0.094 ms, p95 event loop ~16.78 ms.
