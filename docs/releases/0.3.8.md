# ProbRAW 0.3.8

ProbRAW 0.3.8 is a corrective release for a serious preview/rendering bug:
what is adjusted on screen must match the rendered TIFF in color-managed
applications. It keeps the exact render path introduced in 0.3.7 and reduces
its cost with RAW demosaic caching.

## Main Changes

- RAW preview uses the same effective render and color path as export, avoiding
  alternate display paths that could hide color casts or chromatic adjustments
  that later appeared in the TIFF.
- Exact RAW demosaic output is stored in a persistent project cache to speed up
  reloads, exact refinements, single-image export, batch export and the
  development queue without altering the original recipe.
- Cache reads reduce full-frame memory copies on large images.
- The development queue automatically removes successfully rendered items so
  they are not processed again; failed items remain queued with their message.

## Verification

- Full test suite: `330 passed, 1 warning`.
- Focused queue/session suite: `144 passed`.
- Real performance test on an 8288x5520 NEF: first exact render 8.56 s, second
  cached load 2.30 s with the same image mean.
