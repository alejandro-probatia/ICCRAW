# ProbRAW 0.2.2

Release centrada en rendimiento, fluidez de interfaz y reproducibilidad.

## Cambios principales

- `batch-develop` usa multiprocessing real por proceso cuando hay varios
  workers.
- Nueva cache numerica opt-in de demosaico (`use_cache: true`) para reutilizar
  la escena lineal cuando solo cambian ajustes posteriores.
- Nuevos tests golden de hashes canonicos y script de regeneracion controlada.
- Nuevos benchmarks reproducibles de pipeline RAW y de interaccion GUI.
- El refresco final de preview tras arrastrar sliders o curva tonal se encola
  en segundo plano para imagenes grandes cuando no hay preview ICC activo.
- `write_tiff16` reduce temporales NumPy durante la conversion a TIFF16.
- La heuristica automatica de workers reserva mas memoria por proceso
  (`2800 MiB`) para RAWs de alta resolucion.

## Resultados locales de referencia

Benchmark con RAW Nikon D850 8288x5520 de 51,5 MiB en Windows 11, Python 3.12.4
y 32 hilos logicos:

- demosaico `dcb`: ~5,36 s,
- demosaico `amaze`: ~5,57 s,
- cache hit de demosaico `dcb`: ~0,16 s,
- `develop` completo con cache hit, TIFF final + audit: ~1,59 s,
- 4 jobs `dcb` con 4 workers: ~5,97 s frente a ~21,31 s serial.

Benchmark GUI con RAW completo 5520x8288:

- slider brillo: p95 evento UI ~0,053 ms, p95 event loop ~16,72 ms,
- curva tonal: p95 evento UI ~0,094 ms, p95 event loop ~16,78 ms.
