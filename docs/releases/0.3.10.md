# ProbRAW 0.3.10

ProbRAW 0.3.10 is a colorimetric-correction and performance release for
professional high-resolution RAW work at 100%.

## Main Changes

- The application no longer accepts a managed preview without an input profile.
  If no session/image ICC exists, the visible recipe uses a real generic
  profile, ProPhoto RGB by default, and display conversion runs directly from
  that profile to the configured monitor ICC.
- The profiled path avoids standard sRGB shortcuts when a source profile exists.
  The internal sRGB signal is reserved for histogram, clipping and diagnostics,
  not as a replacement for `source ICC -> monitor ICC` management.
- Interactive display works on the visible crop at real scale, updates `QImage`
  regions and keeps 1:1 zoom aligned without preventing magnification above
  100%.
- Repeated ICC conversions use dense 8-bit LUTs generated by LittleCMS and
  cached in RAM/on disk. This is an exact transform cache, not a colorimetric
  simplification.
- Color/contrast sliders respond close to real time, and tone curves reduce
  their cost through tonal-LUT caching and shared RGB quantization before the
  ICC conversions needed by the display and instruments.
- Cache buttons now distinguish correctly between preloading only the selected
  image at 1:1 and preloading the visible directory.
- MTF recalculation is isolated from color/contrast adjustment; sharpness still
  analyses real-resolution data when the user uses that tool.

## Verification

- Full suite: `python -m pytest -q` -> `373 passed`.
- Real GUI benchmark with `G:\ProbRAW-TEST\01_ORG\f_16,0.NEF`, DCB, 100%,
  ProPhoto RGB, monitor ICC and clipping overlay enabled:
  - brightness/normal changes: ~41-44 ms per visible preview;
  - tone curve: ~62 ms per visible preview;
  - full RAW source: `4044 x 6080`, ~281 MB.
